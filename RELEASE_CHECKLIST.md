
# RELEASE CHECKLIST — 每次改版必做

---

## 0) 發布前必做（索引更新流程）

- 【必做】新增 / 修改文章後，先在本機執行：
  - `python3 scripts/build_site_index.py`
- 確認 repo 根目錄的 `site-index.json` 已更新：
  - `buildId` 改變
  - `generatedAt` 更新
  - `posts` 數量合理
  - `folderCounts` 有同步
- 【必做】`site-index.json` 需跟著本次改動一起 commit / push
- 若 push 後有 Actions 重建索引，也必須確認成功

---

## 1) 類別 / 計數索引更新（必做）

- 確認 `site-index.json` 的改動有被提交
- 確認上線後類別 / 計數有更新
- 若數字未更新：
  - 先 Ctrl+F5 或清除站台快取後再確認
- 若索引沒更新，本次改版視為未完成

---

## 2) 檢查類別索引邏輯（必做）

確認以下規則沒有被破壞：

- 類別索引來源是全站文章
- 有 `.post-tag` 的正式文章要進索引
- C++ 類別頁（`scope=cpp-notes`）只顯示：
  - `/cpp-notes/all problems/`
- 其他題單文章即使不在 C++ 類別頁顯示，也不得從全站索引消失

---

## 3) UI / 顯示驗收（必做）

### 3.0 TOC 驗收

- TOC 必須與文章內容同步
- 若目前規則為只收 `h2`：
  - 不可混入 `h1`
  - 不可混入 `h3` / `h4`
- 不可出現文章內不存在的 TOC 項目
- 至少抽查 3 篇文章：
  - TOC 是否完整
  - 點擊跳轉是否正確
  - 強制刷新後是否仍正確

### 3.1 `categories.html`

- 類別數量正確
- 能看到非 C++ 文章
- 相關文章命中規則正常

### 3.2 `category.html`

- `scope=cpp-notes` 時只顯示 `all problems`
- `已顯示 N / 共 M 篇` 規則正確
- 搜尋 / 分頁 / 排序後顯示邏輯不亂掉

### 3.3 搜尋功能

- 大小寫搜尋正常
- 搜尋後 `已顯示 N` 會變
- `共 M` 不會因搜尋而改變

### 3.4 排序功能

- 字母 / 時間切換正常
- 圖示方向正確
- 不會和資料來源衝突

---

## 4) C++ 計數器驗收（必做）

- `cpp.html` 計數不得寫死
- 計數必須來自 `site-index.json`
- `scope=cpp-notes` 只計入 `all problems`

---

## 5) 新增文章時（必做）

- 新文章可正常開啟
- 有正式文章結構：
  - 內容
  - TOC / 對應 TOC 邏輯
  - 文章資訊
  - `.post-tag`
- 若是 C++ 題解文章：
  - 預設必須進 `cpp-notes/all problems/`
- 若 `all problems` 已有同題：
  - 不得重複新增第二篇同題 `all problems`
- 更新完後重新檢查：
  - categories
  - category
  - `cpp.html`
  - `site-index.json`

---

## 6) 題單補收錄驗收（本次新增，必做）

當你把既有題解補進題單資料夾（如 `grind75` / `neetcode150`）時，必須逐項確認：

### 6.1 實體文章頁
- 題單資料夾內真的有對應 html 檔

### 6.2 題單首頁
- 該題單的 `index.html` 有列出該文章
- 不是只有資料夾有檔案、首頁卻沒列

### 6.3 `site-index.json`
- `posts` 內真的有該題條目
- `folderCounts` 也同步增加
- 不可只改 count、不改 posts

### 6.4 頁面內分類語意
- breadcrumb 正確
- permalink / 原文連結正確
- 題單名稱正確
- 不殘留 `all problems` / `全部題單文章` / 舊資料夾字樣

### 6.5 正文一致性
- 與 `all problems` 版本正文一致
- 不可因為換資料夾就刪節或改寫內容

---

## 7) 題單首頁排序驗收（本次新增，必做）

### 7.1 `all problems`
- 依題號數字遞增

### 7.2 指定題單（如 `grind75` / `neetcode150`）
- 依我指定的原始清單順序
- 只收部分題目時，也必須維持原始順序中的相對位置
- 不可依新增時間亂排
- 不可把新補題目直接加在最後

### 7.3 抽查方式
至少抽查：

- 第一題
- 中間一題
- 新補的一題
- 最後一題

確認其相對位置正確。

---

## 8) 三一致驗收（本次新增，必做）

對每個有改動的題單資料夾，都要檢查以下三者一致：

1. **實體文章檔案數**
2. **該資料夾 `index.html` 卡片數**
3. **`site-index.json` 的 folder count**

只要不一致，本次改版就不算完成。

---

## 9) 頁首註解 / 模板殘留驗收（本次新增，必做）

對於新複製或新建立的文章頁，至少抽查數篇：

- 第一行註解是否統一
- 是否符合當前資料夾語意
- 是否殘留舊資料夾名稱
- 是否有的有註解、有的沒註解、有的亂寫題名

若同一資料夾內格式不一致，需修正後再發布。

---

## 10) 發布前最後人工抽查（必做）

至少抽查以下頁面各 1～3 篇：

- `all problems`
- `grind75`
- `neetcode150`
- 其他本次有改的題單

每篇至少看：

- 頁首模板
- breadcrumb
- 內文是否正常
- TOC 是否正常
- 文章資訊是否完整
- tags 是否正常
- permalink 是否正確

---

## 11) 若本次修的是「會重複踩到的結構性問題」（必做）

若這次修的是以下類型問題之一：

- 題單首頁漏列文章
- `site-index.json` 不同步
- 題單排序被亂排
- 複製頁面後 breadcrumb / 註解殘留
- TOC 與內容不同步
- 計數器與實體檔案數不一致

則發布前必須確認：

- `SITE_RULES.md` 已補入對應法律，或
- `RELEASE_CHECKLIST.md` 已補入對應驗收步驟

只修檔案、不修流程，視為修正不完整。
