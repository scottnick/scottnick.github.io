1888. Minimum Number of Flips to Make the Binary String Alternating

練習日期：2026-03-07

難度：Medium

類型：String、Dynamic Programming、Sliding Window

📘 題目敘述

給你一個二元字串 s。你可以對字串進行任意次、任意順序的以下兩種操作：

  • Type-1：移除字串 s 開頭的字元，並把它接到字串尾端
  • Type-2：選擇字串中的任意一個字元並翻轉它，也就是如果它是 '0' 就變成 '1'，如果它是 '1' 就變成 '0'

請回傳最少需要做幾次 Type-2 操作，才能讓 s 變成 alternating。

如果一個字串中沒有任何兩個相鄰字元相同，就稱這個字串是 alternating。例如 "010" 和 "1010" 是 alternating，而 "0100" 不是。

條件限制

  • 1 <= s.length <= 10^5
  • s[i] 只會是 '0' 或 '1'

🧠 解題思路

這題雖然有兩種操作，但核心困難在 Type-1（旋轉）。

關鍵觀察是 alternating 字串只有兩種型態：

  • 010101...
  • 101010...

所以我們只需要知道「目前字串和這兩種型態各差多少位」，最少 flip 就是兩者較小值。

所有變數

  • c0：目前字串相對於 010101... 的 mismatch 數量
  • c1：目前字串相對於 101010... 的 mismatch 數量
  • n：字串長度
  • ans：所有旋轉情況中的最小 flip 次數

🪜 主要流程步驟

1. 先掃一次字串，算出 c0 與 c1

逐位比對兩種 alternating 型態，統計 mismatch 數量。

2. 偶數長度可直接回傳

若 n 是偶數，旋轉後奇偶結構不會本質互換，答案就是 min(c0, c1)。

3. 奇數長度要模擬每次旋轉

若 n 是奇數，每轉一次，所有位置的奇偶角色對調，所以每次都要更新 c0 與 c1。

4. 用 O(1) 轉移更新 mismatch 計數

把前端字元移到尾端時，不必重掃整串，只要根據該字元原本的匹配情況做常數時間調整：

if (s[i] - '0' == i % 2) {
    c1++;
    c0--;
} else {
    c0++;
    c1--;
}

5. 每次旋轉都更新最小答案

每次更新完計數後，取 min(c0, c1)，再與全域答案比較。

6. 走完全部旋轉，回傳 ans

跑完 n 次後，等於考慮過所有 possible rotation，回傳最小值即可。

⏱️ 複雜度

  • 時間複雜度：O(n)
  • 空間複雜度：O(1)

💻 程式碼實作

class Solution {
public:
    int minFlips(string s) {
        int c0 = 0, c1 = 0;
        int n = s.size();

        for (int i = 0; i < n; i++) {
            if (s[i] - '0' == i % 2) {
                c1++;
            } else {
                c0++;
            }
        }

        if (n % 2 == 0) {
            return min(c0, c1);
        }

        int ans = min(c0, c1);
        for (int i = 0; i < n; i++) {
            if (s[i] - '0' == i % 2) {
                c1--;
                c0++;
            } else {
                c0--;
                c1++;
            }
            ans = min(ans, min(c0, c1));
        }

        return ans;
    }
};
